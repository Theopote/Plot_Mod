# Plot 道路插件问题修复总结

修复日期: 2026-07-13

## 已完成的修复（5个严重问题）

### ✅ 1. 修复数据丢失问题 - RoadPersistenceManager.java

**问题**: 加载网络失败时会清空历史记录，违反原子性原则

**修复内容**:
- 在 `loadNetworkFile()` 方法中调整了执行顺序
- 只有在 `RoadNetwork.loadFrom()` 成功后才清空历史
- 添加了注释说明原子性保证

**修改文件**: `src/main/java/com/plot/plugin/road/manager/RoadPersistenceManager.java`

**修改位置**: 第83-99行

**影响**: 保护用户的撤销/重做历史，防止因加载失败导致数据丢失

---

### ✅ 2. 修复并发安全问题 - RoadNetwork.java

**问题**: 使用非线程安全的 `LinkedHashMap`，在多线程环境下可能导致 `ConcurrentModificationException`

**修复内容**:
- 将 `nodes`、`edges`、`roads` 三个 Map 从 `LinkedHashMap` 改为 `ConcurrentHashMap`
- 更新了类注释，说明线程安全保证
- 修改了 import 语句

**修改文件**: `src/main/java/com/plot/plugin/road/model/RoadNetwork.java`

**修改位置**: 
- 第28行: 添加 `import java.util.concurrent.ConcurrentHashMap;`
- 第34-41行: 更新类注释和字段声明

**影响**: 
- 支持多线程并发访问（UI线程、渲染线程、持久化线程）
- 消除 `ConcurrentModificationException` 风险
- 提高多线程环境下的稳定性

**注意**: `ConcurrentHashMap` 不保证迭代顺序，如果需要保持插入顺序，后续可能需要使用其他同步机制

---

### ✅ 3. 修复资源泄漏 - RoadSystemPlugin.java

**问题**: `onEnable()` 中事件监听器订阅后，如果后续操作失败，监听器不会被清理

**修复内容**:
- 将事件订阅和网络加载包装在 try-catch 块中
- 如果 `loadForCurrentProject()` 失败，自动取消订阅监听器
- 重新抛出异常，确保错误不被掩盖

**修改文件**: `src/main/java/com/plot/plugin/RoadSystemPlugin.java`

**修改位置**: 第57-100行 (`onEnable()` 方法)

**影响**: 
- 防止内存泄漏
- 确保插件禁用后不再接收事件
- 避免已销毁对象的方法调用

---

### ✅ 4. 修复空指针风险 - RoadGenerator.java

**问题**: 存在 TOCTOU（Time-of-Check-Time-of-Use）问题，`client.world` 可能在检查后、使用前变为 null

**修复内容**:
- 分离 `client` 和 `world` 的 null 检查
- 先缓存 `world` 引用，再检查是否为 null
- 使用缓存的引用而非重复访问 `client.world`
- 改进了错误日志，分别提示客户端和世界实例的问题

**修改文件**: `src/main/java/com/plot/plugin/road/RoadGenerator.java`

**修改位置**: 第96-106行

**影响**: 
- 消除了多线程竞态条件
- 在玩家切换维度或退出世界时更加稳定
- 减少 `NullPointerException` 崩溃风险

---

### ✅ 5. 修复数据一致性问题 - RoadNetworkManager.java

**问题**: `deleteEdge()` 删除边后，选择状态可能指向已删除的对象

**修复内容**:
- 在 `deleteEdge()` 方法末尾添加 `ensureSelectionValid()` 调用
- 添加注释说明为何需要验证选择状态

**修改文件**: `src/main/java/com/plot/plugin/road/manager/RoadNetworkManager.java`

**修改位置**: 第209-220行

**影响**: 
- 确保删除边后选择状态保持有效
- 防止 UI 显示错误或后续操作失败
- 消除潜在的 `NullPointerException`

## 待修复的问题

- 位置: `hasOverlappingOverride()` 方法
- 问题: 无效区间（startDistance >= endDistance）仍参与比较
- 建议: 先验证区间有效性

**9. 浮点数比较问题 - RoadGenerator.java**
- 位置: `findHeightInfo()` 和 `sameSegment()` 方法
- 问题: 累积误差可能导致段匹配失败
- 建议: 使用引用相等或段索引

**10. 异常处理不当 - RoadNetworkManager.java**

- 问题: 捕获所有异常继续处理，可能掩盖严重问题
