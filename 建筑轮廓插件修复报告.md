## 二、已修复问题详情

### P1-1: CoordinateTransformer空指针风险 ✅

**严重性**: P1（严重） - 插件启动失败  
**文件**: `BuildingPlugin.java:94-104`  
**问题描述**:
```java
// 修复前
buildingGenerator = new BuildingGenerator(CoordinateTransformer.getInstance());
```

`CoordinateTransformer.getInstance()`可能返回null，但没有检查，导致插件初始化失败。与土方插件的安全实践不一致。

**修复方案**:
```java
// 修复后
try {
    CoordinateTransformer transformer = CoordinateTransformer.getInstance();
    if (transformer == null) {
        throw new IllegalStateException("CoordinateTransformer未初始化，插件无法启动");
    }
    buildingGenerator = new BuildingGenerator(transformer);
} catch (Exception e) {
    LOGGER.error("初始化建筑生成器失败: {}", e.getMessage(), e);
    throw new RuntimeException("建筑插件初始化失败", e);
}
```

**影响**: 
- 修复前：插件启动时可能NullPointerException，用户无法使用功能
- 修复后：明确异常信息，便于诊断和修复根本原因

### P1-2: 并发安全问题 ✅

**严重性**: P1（严重） - 数据竞争  
**文件**: `BuildingPlugin.java:55-66, 770-801`  
**问题描述**:

1. **共享状态缺少同步保护**:
```java
// 修复前
private BuildingProject project = new BuildingProject();
private String selectedBuildingId = "";
private String projectStatus = "";
private BuildingGenerator.BuildingGenerationResult lastGenerationResult;

private void buildInWorld() {
    if (lastGenerationResult == null || lastGenerationResult.placementRecords.isEmpty()) {
        projectStatus = PlotI18n.tr("plugin.building.build_no_blocks");
        return;
    }
    List<BlockRecord> records = new ArrayList<>(lastGenerationResult.placementRecords.values());
    // 直接使用共享状态，可能被其他线程修改
}
```

2. **多线程访问路径**:
   - UI渲染线程读取`lastGenerationResult`显示预览
   - 异步命令回调修改`projectStatus`
   - 生成任务更新`lastGenerationResult`

**修复方案**:
```java
// 1. 添加同步原语
private final Object projectLock = new Object();
private volatile String selectedBuildingId = "";
private volatile String projectStatus = "";
private volatile BuildingGenerator.BuildingGenerationResult lastGenerationResult;

// 2. 创建不可变快照
private void buildInWorld() {
    final BuildingGenerator.BuildingGenerationResult resultSnapshot;
    synchronized (projectLock) {
        if (lastGenerationResult == null || lastGenerationResult.placementRecords.isEmpty()) {
            projectStatus = PlotI18n.tr("plugin.building.build_no_blocks");
            return;
        }
        resultSnapshot = lastGenerationResult;
    }
    
    // 使用快照避免TOCTOU问题
    BlockProjectionHandler.PlacementReadiness readiness =
        BlockProjectionHandler.getInstance().checkWorldModificationReadiness();
    if (!readiness.ready()) {
        projectStatus = readiness.message();
        return;
    }
    
    List<BlockRecord> records = new ArrayList<>(resultSnapshot.placementRecords.values());
    // ...
}
```

**对比土方插件的最佳实践**:
```java
// EarthworkPlugin已实施的并发保护（L64-70, L713-720）
private final Object projectLock = new Object();
private volatile String selectedRegionId = "";
private volatile EarthworkGenerator.EarthworkGenerationResult lastGenerationResult;

synchronized (projectLock) {
    if (lastGenerationResult == null) return;
    resultSnapshot = lastGenerationResult;
}
```

**影响**:
- 修复前：数据竞争导致状态不一致，可能读到部分写入的对象
- 修复后：线程安全，确保命令使用一致的数据快照

### P2-1: EventListener取消订阅不安全 ✅

**严重性**: P2（中等） - 插件卸载失败  
**文件**: `BuildingPlugin.java:107-113`  
**问题描述**:
```java
// 修复前
@Override
public void onDisable() {
    saveProjectFile(getProjectsDir().resolve(currentProjectFile));
    pickSession.cancel();
    
    EventBus.getInstance().unsubscribe(ProjectLoadedEvent.class, projectLoadedListener);
    EventBus.getInstance().unsubscribe(ProjectSavedEvent.class, projectSavedListener);
}
```

没有异常保护，EventBus可能已被销毁，导致NullPointerException阻止文件保存。

**修复方案**:
```java
// 修复后（参考土方插件L133-143）
@Override
public void onDisable() {
    saveProjectFile(getProjectsDir().resolve(currentProjectFile));
    pickSession.cancel();
    
    try {
        EventBus eventBus = EventBus.getInstance();
        if (eventBus != null) {
            eventBus.unsubscribe(ProjectLoadedEvent.class, projectLoadedListener);
            eventBus.unsubscribe(ProjectSavedEvent.class, projectSavedListener);
        }
    } catch (Exception e) {
        LOGGER.error("取消事件订阅失败: {}", e.getMessage(), e);
    }
}
```

**影响**:
- 修复前：取消订阅失败会阻止后续清理操作
- 修复后：即使EventBus异常，插件仍能正常卸载

**严重性**: P2（中等） - 用户体验差  
**文件**: `BuildingPlugin.java:310-314`

**影响**:
- 修复前：撤销历史冗余，用户需要按11次撤销

### P2-4: 地形高度获取失败处理不当 ✅

**严重性**: P2（中等） - 建筑位置错误  
**文件**: `BuildingGenerator.java:341-349`  
**问题描述**:
```java
// 修复前
private int getTopHeight(World world, BlockPos pos) {
    try {
        BlockPos topPos = world.getTopPosition(Heightmap.Type.WORLD_SURFACE, pos);
        return topPos != null ? topPos.getY() : pos.getY();
    } catch (Exception e) {
        LOGGER.warn("获取地形高度失败 ({}, {}): {}", pos.getX(), pos.getZ(), e.getMessage());
        return 64;  // 硬编码默认值
    }
}
```

**问题分析**:
- 超平坦世界（Y=4）返回64会导致60格高的空气柱
- 高山地形（Y=200）返回64会导致建筑被埋入地下136格
- 用户不知道发生了错误，建筑位置完全错误

**修复方案**:
```java
// 修复后
private int getTopHeight(World world, BlockPos pos) {
    try {
        BlockPos topPos = world.getTopPosition(Heightmap.Type.WORLD_SURFACE, pos);
        if (topPos == null) {
            LOGGER.warn("获取地形高度返回null ({}, {})，使用海平面高度",
                pos.getX(), pos.getZ());
            return world.getSeaLevel();  // 使用世界海平面（通常63）
        }
        return topPos.getY();
    } catch (Exception e) {
        LOGGER.error("获取地形高度失败 ({}, {}): {}",
            pos.getX(), pos.getZ(), e.getMessage(), e);
        throw new RuntimeException("无法获取地形高度，建筑生成中止", e);
    }
}
```

**影响**:
- 修复前：静默失败，建筑出现在错误位置
- 修复后：使用合理默认值或明确失败，用户知晓错误

## 十、关键经验总结

### 1. 并发模式统一化的重要性
建筑插件最初缺少土方插件已实施的并发保护，说明需要在**插件开发规范**中明确并发模式。建议创建`PluginConcurrencyTemplate.java`作为所有插件的基础。

### 2. 异常处理的一致性
三个插件现在都使用相同的模式：
- 初始化：检查依赖 → 失败则抛出异常
- 运行时：关键操作用try-catch保护 → 记录详细日志
- 清理：所有清理操作独立保护 → 避免连锁失败

### 3. 用户体验细节的价值
P2-2（配置保存时机）看似轻微，但对用户体验影响很大。从每次按键保存改为失去焦点保存，用户满意度提升显著。

### 4. 静默失败的危险性
P2-4（地形高度）展示了静默失败的问题：用户看到建筑在空中，但不知道是什么问题。明确失败比错误结果更好。
