# 建筑轮廓插件（BuildingPlugin）全面检查报告

**检查日期**: 2026-07-13  
**插件名称**: BuildingPlugin（建筑轮廓插件）  
**代码规模**: 约2000行核心代码，15个Java文件  
**发现问题**: 11个（2个P1级，5个P2级，4个P3级）

---

## 一、插件概览

### 1.1 核心功能模块

建筑轮廓插件是Plot项目的第三个核心插件，负责生成建筑结构。功能包括：

- **轮廓管理**: 从Shape认领建筑轮廓（`BuildingFootprintPickSession`）
- **参数编辑**: 层高、材质、窗门、屋顶配置（`BuildingFootprint`）
- **生成引擎**: 墙体/楼板/屋顶/地基生成（`BuildingGenerator`, `BuildingRoofGenerator`）
- **几何工具**: 多边形偏移、采样、坐标转换（`BuildingGeometryUtils`）
- **历史管理**: JSON快照撤销/重做（`BuildingProjectHistory`）
- **命令系统**: 异步方块放置与撤销（`BuildingGenerateCommand`）

### 1.2 架构设计评估

**优点**:
- ✅ 良好的分层架构，职责清晰
- ✅ 使用CopyOnWriteArrayList保护事件监听器
- ✅ 大量使用`List.copyOf()`防止外部修改
- ✅ 纯函数设计（BuildingFoundationUtils全静态）
- ✅ 完善的输入验证（所有setter都有范围限制）

**问题**:
- ⚠️ 并发安全不足（缺少对共享状态的保护）
- ⚠️ 部分异常处理薄弱
- ⚠️ 与土方插件的安全模式不一致

#### **P1-1: CoordinateTransformer空指针风险**

**位置**: `BuildingPlugin.java:96`  
**严重性**: P1 - 插件启动失败

**问题代码**:
```java
@Override
public void onEnable() {
    try {
        buildingGenerator = new BuildingGenerator(CoordinateTransformer.getInstance());
    } catch (Exception e) {
        LOGGER.error("初始化建筑生成器失败: {}", e.getMessage(), e);
    }
}
```

**问题分析**:
- `getInstance()`可能返回null，但没有检查
- 即使捕获异常，`buildingGenerator`仍为null，后续调用会崩溃
- 与土方插件的安全实践不一致

**对比土方插件的安全实践**:
```java
// EarthworkPlugin.java:112-120
CoordinateTransformer transformer = CoordinateTransformer.getInstance();
if (transformer == null) {
    throw new IllegalStateException("CoordinateTransformer未初始化，插件无法启动");
}
earthworkGenerator = new EarthworkGenerator(transformer);
```

**修复建议**:
```java
try {
    CoordinateTransformer transformer = CoordinateTransformer.getInstance();
    if (transformer == null) {
        throw new IllegalStateException("CoordinateTransformer未初始化，插件无法启动");
    }
    buildingGenerator = new BuildingGenerator(transformer);
} catch (Exception e) {
    LOGGER.error("初始化建筑生成器失败: {}", e.getMessage(), e);
    throw new RuntimeException("建筑插件初始化失败", e);
}
```

**影响**: 插件启动时可能NullPointerException，用户无法使用功能

#### **P1-2: 并发安全问题 - 缺少同步保护**

**位置**: `BuildingPlugin.java:55-66, 775-786`  
**严重性**: P1 - 数据竞争

**问题代码**:
```java
private BuildingProject project = new BuildingProject();  // L55
private String selectedBuildingId = "";  // L61
private String projectStatus = "";  // L62
private BuildingGenerator.BuildingGenerationResult lastGenerationResult;  // L65

private void buildInWorld() {
    if (lastGenerationResult == null || lastGenerationResult.placementRecords.isEmpty()) {
        projectStatus = PlotI18n.tr("plugin.building.build_no_blocks");
        return;
    }
    // 直接使用共享状态，没有同步保护
    List<BlockRecord> records = new ArrayList<>(lastGenerationResult.placementRecords.values());
}
```

**问题分析**:
1. **共享状态无保护**: `project`, `lastGenerationResult`, `projectStatus`被多个线程访问
2. **多线程访问路径**:
   - UI渲染线程读取`lastGenerationResult`显示预览
   - 异步命令回调修改`projectStatus`
   - 生成任务更新`lastGenerationResult`
3. **TOCTOU问题**: 检查null后使用前，状态可能被其他线程修改
4. **没有volatile修饰**: 线程间可能读取到过期值

**对比土方插件的并发保护**:
```java
// EarthworkPlugin.java:64-70
private final Object projectLock = new Object();
private volatile String selectedRegionId = "";
private volatile String projectStatus = "";
private volatile EarthworkGenerator.EarthworkGenerationResult lastGenerationResult;

// L713-720
synchronized (projectLock) {
    if (lastGenerationResult == null || lastGenerationResult.placementRecords.isEmpty()) {
        projectStatus = PlotI18n.tr("plugin.earthwork.build_no_blocks");
        return;
    }
    resultSnapshot = lastGenerationResult;
}
```

**修复建议**:
```java
// 1. 添加同步原语
private final Object projectLock = new Object();
private volatile String selectedBuildingId = "";
private volatile String projectStatus = "";
private volatile BuildingGenerator.BuildingGenerationResult lastGenerationResult;

// 2. 创建不可变快照
private void buildInWorld() {
    final BuildingGenerator.BuildingGenerationResult resultSnapshot;
    synchronized (projectLock) {
        if (lastGenerationResult == null || lastGenerationResult.placementRecords.isEmpty()) {
            projectStatus = PlotI18n.tr("plugin.building.build_no_blocks");
            return;
        }
        resultSnapshot = lastGenerationResult;
    }
    
    BlockProjectionHandler.PlacementReadiness readiness =
        BlockProjectionHandler.getInstance().checkWorldModificationReadiness();
    if (!readiness.ready()) {
        projectStatus = readiness.message();
        return;
    }
    
    List<BlockRecord> records = new ArrayList<>(resultSnapshot.placementRecords.values());
    // ...
}
```

**影响**: 数据竞争导致状态不一致，可能读取到部分写入的对象

#### **P2-4: 地形高度获取失败处理不当**

**位置**: `BuildingGenerator.java:341-349`  
**严重性**: P2 - 建筑位置错误

**问题代码**:
```java
private int getTopHeight(World world, BlockPos pos) {
    try {
        BlockPos topPos = world.getTopPosition(Heightmap.Type.WORLD_SURFACE, pos);
        return topPos != null ? topPos.getY() : pos.getY();
    } catch (Exception e) {
        LOGGER.warn("获取地形高度失败 ({}, {}): {}", pos.getX(), pos.getZ(), e.getMessage());
        return 64;  // 硬编码默认值
    }
}
```

**问题分析**:
- 返回硬编码的64，可能与实际地形相差很远
- 超平坦世界（Y=4）会导致60格高的空气柱
- 高山地形（Y=200）会导致建筑被埋入地下
- 失败时仍继续生成，用户不知道发生了错误

**修复建议**:
```java
private int getTopHeight(World world, BlockPos pos) {
    try {
        BlockPos topPos = world.getTopPosition(Heightmap.Type.WORLD_SURFACE, pos);
        if (topPos == null) {
            LOGGER.warn("获取地形高度返回null ({}, {})，使用海平面高度", 
                pos.getX(), pos.getZ());
            return world.getSeaLevel();  // 使用世界海平面
        }
        return topPos.getY();
    } catch (Exception e) {
        LOGGER.error("获取地形高度失败 ({}, {}): {}", 
            pos.getX(), pos.getZ(), e.getMessage(), e);
        throw new RuntimeException("无法获取地形高度，建筑生成中止", e);
    }
}
```

**影响**: 建筑出现在错误位置，用户体验差

## 八、关键建议

### 1. 统一并发模式
建议创建`PluginConcurrencyTemplate.java`作为所有插件的基础，统一：
- 共享状态保护（projectLock + volatile）
- 快照模式（避免TOCTOU）
- 异步回调的线程安全

### 2. 统一异常处理
三个插件应使用相同的模式：
- 初始化：检查依赖 → 失败则抛出异常
- 运行时：关键操作用try-catch保护 → 记录详细日志
- 清理：所有清理操作独立保护 → 避免连锁失败

### 3. 改进测试覆盖
建议添加：
- 并发场景测试（快速切换+异步操作）
- 边界条件测试（极端地形高度）
- 异常路径测试（依赖未初始化）
