## 问题清单

### 🔴 严重问题（1个）

#### S1. 填土系数应用不一致 ⚠️ **核心功能错误**

**位置**: `EarthworkGenerator.java:114-118`

**问题描述**:
```java
} else if (groundY < targetElevation) {
    // 填方：实际放置方块数
    for (int y = groundY + 1; y <= targetElevation; y++) {
        BlockPos pos = new BlockPos(column.getX(), y, column.getZ());
        recordBlock(result, pos, fillBlockId, ChangeType.FILL, projectionHandler);
    }
    // 累加填方体积（未考虑填土系数）
    result.fillVolume += targetElevation - groundY;
}
```

**相关代码**: `EarthworkBalanceUtils.java:58-61`
```java
// 在计算平衡标高时考虑了填土系数
double fillVolumeAtHeight = 0.0;
for (int sample : samples) {
    if (sample < candidateHeight) {
        fillVolumeAtHeight += (candidateHeight - sample) * fillFactor; // 应用了系数
    }
}
```

**问题**: 
- 在计算平衡标高时，填方体积乘以了填土系数（`fillFactor`，默认1.1）
- 但在实际生成方块时，填方体积没有应用系数
- 导致实际生成的填方体积与平衡计算时的填方体积不一致

**影响**: 🔴 **严重** - 核心功能逻辑错误
- 用户期望挖方量和填方量平衡（考虑松散系数）
- 但实际生成后，填方体积会比预期少约10%（假设系数1.1）
- 这违背了"土方平衡"的基本目标

**示例**:
- 假设平衡标高计算结果：挖方100m³，填方110m³（系数1.1后）
- 实际生成：挖方100m³，填方100m³（未应用系数）
- 结果：不平衡！

**修复方案**:

**方案A**（推荐）: 移除计算时的填土系数应用
```java
// EarthworkBalanceUtils.java:58-61
double fillVolumeAtHeight = 0.0;
for (int sample : samples) {
    if (sample < candidateHeight) {
        // 不应用填土系数，保持方块数一致
        fillVolumeAtHeight += (candidateHeight - sample);
    }
}
```

**方案B**: 在生成时也应用填土系数（但实际放置困难）
- 需要按比例减少放置的填方方块数
- 实现复杂，且可能导致表面不平整

**优先级**: 🔴 **P0 - 立即修复**

### 🔴 严重问题（代码质量）

#### S2. 空指针风险 - CoordinateTransformer

**位置**: `EarthworkPlugin.java:111`

**问题**:
```java
earthworkGenerator = new EarthworkGenerator(CoordinateTransformer.getInstance());
```

`CoordinateTransformer.getInstance()` 可能返回null，但没有检查。

**影响**: 插件初始化失败，后续所有操作崩溃

**修复**:
```java
CoordinateTransformer transformer = CoordinateTransformer.getInstance();
if (transformer == null) {
    throw new IllegalStateException("CoordinateTransformer未初始化");
}
earthworkGenerator = new EarthworkGenerator(transformer);
```

**优先级**: 🔴 **P0**

---

#### S3. 并发安全问题

**位置**: `EarthworkPlugin.java:59-68`

**问题**:
```java
private EarthworkProject project = new EarthworkProject();
private final List<Shape> selectedRegions = new ArrayList<>();
private String selectedRegionId = "";
```

这些字段在UI线程和异步方块放置任务间共享，但没有同步保护。

**影响**: 在方块放置期间修改project可能导致数据不一致或崩溃

**修复**: 使用`synchronized`或创建不可变快照传递给后台任务

**优先级**: 🔴 **P0**

### 🟡 中等问题（8个）

#### M1. 资源泄漏 - EventListener

**位置**: `EarthworkPlugin.java:127-128`

**问题**: 如果`EventBus.getInstance()`在disable时返回null，监听器不会被清理

**修复**:
```java
try {
    EventBus eventBus = EventBus.getInstance();
    if (eventBus != null) {
        eventBus.unsubscribe(ProjectLoadedEvent.class, projectLoadedListener);
        eventBus.unsubscribe(ProjectSavedEvent.class, projectSavedListener);
    }
} catch (Exception e) {
    LOGGER.error("取消事件订阅失败", e);
}
```

**优先级**: 🟡 **P1**
