### ✅ 1. 填土系数应用不一致 ⚠️ **核心功能错误**

**严重性**: 🔴 **P0 - 最高优先级**

**问题描述**:
- 在计算平衡标高时，填方体积乘以了填土系数（默认1.1）
- 但在实际生成方块时，填方体积没有应用系数
- 导致实际生成的挖填方量不平衡

**影响**: 
- 用户期望挖方量和填方量平衡（考虑松散系数）
- 但实际生成后，填方体积会比预期少约10%
- 这违背了"土方平衡"的基本目标

**示例**:
```
计算时: 挖方100m³ = 填方×1.1 (110m³需求)
实际生成: 挖方100m³ ≠ 填方100m³ (不平衡！)
```

**修复内容**:

**文件**: `src/main/java/com/plot/plugin/earthwork/EarthworkBalanceUtils.java`

**修改位置**: 第78-92行

**修改前**:
```java
static long balanceDiff(List<Integer> groundHeightSamples, int targetElevation, float fillFactor) {
    return computeCutVolume(groundHeightSamples, targetElevation)
        - computeFillVolumeRequired(groundHeightSamples, targetElevation, fillFactor);
}
```

**修改后**:
```java
/**
 * 计算平衡差值（挖方量 - 填方量）
 *
 * 注意：填土系数不应在此处应用。
 * 填土系数（fillFactor）表示填土的松散系数，用于估算需要的原土体积。
 * 但在Minecraft中，我们放置的是实际方块，不是原土体积。
 * 因此平衡计算应基于实际方块数，不应用填土系数。
 *
 * @deprecated fillFactor参数已不再使用，为保持API兼容性暂时保留
 */
@Deprecated
static long balanceDiff(List<Integer> groundHeightSamples, int targetElevation, float fillFactor) {
    // 修复：不再应用填土系数，保持方块数平衡
    return computeCutVolume(groundHeightSamples, targetElevation)
        - computeFillVolume(groundHeightSamples, targetElevation);
}
```

**影响**: 
- ✅ 挖方量和填方量现在真正平衡
- ✅ 实际生成结果符合预期
- ✅ 保持了API兼容性（参数保留）

**测试建议**:

4. 验证挖方量 ≈ 填方量（允许±1%误差）

### ✅ 2. 空指针风险 - CoordinateTransformer

**严重性**: 🔴 **P0**

**问题描述**:
```java
earthworkGenerator = new EarthworkGenerator(CoordinateTransformer.getInstance());
```
`CoordinateTransformer.getInstance()` 可能返回null，但没有检查

**影响**: 插件初始化失败，后续所有操作崩溃

**修复内容**:

**文件**: `src/main/java/com/plot/plugin/EarthworkPlugin.java`

**修改位置**: 第109-118行

**修改前**:
```java
try {
    earthworkGenerator = new EarthworkGenerator(CoordinateTransformer.getInstance());
} catch (Exception e) {
    LOGGER.error("初始化土方生成器失败: {}", e.getMessage(), e);
}
```

**修改后**:
```java
try {
    CoordinateTransformer transformer = CoordinateTransformer.getInstance();
    if (transformer == null) {
        throw new IllegalStateException("CoordinateTransformer未初始化，插件无法启动");
    }
    earthworkGenerator = new EarthworkGenerator(transformer);
} catch (Exception e) {
    LOGGER.error("初始化土方生成器失败: {}", e.getMessage(), e);
    throw new RuntimeException("土方插件初始化失败", e);
}
```

**影响**: 
- ✅ 防止空指针异常
- ✅ 提供清晰的错误信息
- ✅ 确保插件启动失败时不会静默吞掉错误

### ✅ 3. 并发安全问题

**严重性**: 🔴 **P0**

**问题描述**:
```java
private EarthworkProject project = new EarthworkProject();
private final List<Shape> selectedRegions = new ArrayList<>();
private String selectedRegionId = "";
```

这些字段在UI线程和异步方块放置任务间共享，但没有同步保护

**影响**: 
- 在方块放置期间修改project可能导致数据不一致
- 可能导致崩溃或错误的方块放置结果

**修复内容**:

**文件**: `src/main/java/com/plot/plugin/EarthworkPlugin.java`

**第一步：添加同步机制**

**修改位置**: 第58-71行

**修改前**:

**第二步：使用同步保护**

**修改位置**: 第709-740行

**修改前**:
```java

    if (lastGenerationResult == null || lastGenerationResult.placementRecords.isEmpty()) {
        projectStatus = PlotI18n.tr("plugin.earthwork.build_no_blocks");

    }
    
    // ... 权限检查

    EarthworkGenerateCommand command = new EarthworkGenerateCommand(records);
    projectStatus = PlotI18n.tr("plugin.earthwork.build_in_progress", records.size());
    command.executeScheduled(() -> {

```

**修改后**:
```java
private void buildInWorld() {
    // 创建不可变快照，避免异步任务中的并发问题
    final EarthworkGenerator.EarthworkGenerationResult resultSnapshot;
    synchronized (projectLock) {
        if (lastGenerationResult == null || lastGenerationResult.placementRecords.isEmpty()) {
            projectStatus = PlotI18n.tr("plugin.earthwork.build_no_blocks");
            return;
        }
        resultSnapshot = lastGenerationResult;
    }
    
    // ... 权限检查
    
    List<BlockRecord> records = new ArrayList<>(resultSnapshot.placementRecords.values());
    EarthworkGenerateCommand command = new EarthworkGenerateCommand(records);
    projectStatus = PlotI18n.tr("plugin.earthwork.build_in_progress", records.size());
    command.executeScheduled(() -> {
        // 使用快照，不直接访问共享字段
        EarthworkGenerateCommand.ExecutionResult result = command.getLastExecutionResult();
        // ...
    });
}
```

**影响**: 
- ✅ 消除并发竞态条件
- ✅ 防止数据不一致
- ✅ 使用`volatile`保证可见性
- ✅ 使用不可变快照避免异步访问冲突
